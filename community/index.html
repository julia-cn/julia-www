请花一点时间阅读Julia社区标准。我们希望您参与任何与Julia相关的在线或离线论坛都能遵守这些标准。


官方频道


GitHub

我们使用GitHub来作为Julia本身的项目管理工具。在那里，我们托管我们的源代码，跟踪问题，并接受拉取请求（pr）。我们使用问题跟踪器进行bug报告、特性请求和提议的更改。如需帮助支持和提问，请使用Discourse。

Discourse

Julia的主要在线讨论场所是Discourse 论坛。在那里可以进行的讨论场景如下：

  • 不断更新Julia的重要公告，如语言新版本的发布 和 JuliaCon。
  • 提出问题 询问有关使用或开发Julia或其相关包和工具的任何方面的问题。Discourse 论坛是最佳的场所，你的Julia的问题会获得快速，高质量的答案。有不同的Julia应用领域的子类，如机器学习，金融，可视化和GPU。新用户的问题总是受欢迎的：您可以使用“FirstSteps”标记来突出显示您的问题，以便其他新用户看到后也会受益。
  • 回答别人的问题：当人们分享他们的知识和经验时，我们的社会就会繁荣起来。
  • 发布一个新的包 或者征求关于你一直在开发的工具、包或者类库的用户反馈。
  • 广告 Julia相关的工作和社区活动。

对于中国用户，社区也主持简体中文版话语。请点击此Julia中文论坛链接进入讨论。

2016年底，Discourse 论坛取代了Julia的邮件列表，其中最大的两个邮件列表是Julia-users和julia-dev。这些列表现在是只读的，但是您可能希望搜索它们，看看您的问题是否曾经已被提出。

Slack

对于想随意和迅速的谈话，和讨论非正式的问题，我们也提供一个官方的Julia Slack的选项。Slack也是一个很好的选择，如果你认为你需要帮助但不太确定你应该问什么或在哪里问它。如果想要加入，请访问slackinvite.julialang.org以同意社区行为准则并收到邀请。

YouTube

所有JuliaCon视频和其他社区普遍感兴趣的视频都上传到了 Julia 的 YouTube频道。


其他渠道

在Twitter上，可以用 #julialang hash标签发推文。

Julia 还在 julia-lang 标签下的 Stack Overflow 和 julia 下的 Stack Overflow 和 Español 上出现。

对于那些喜欢 IRC（互联网中继聊天）的人来说，Freenode上有一个#julia频道。我们还有一个非官方的Gitter频道。有一个小 Julia subreddit 进行随意的讨论。


国际社区

Julia GitHub小组

  • JuliaInterop – 提升Julia和非Julia之间的兼容性
  • JuliaIO – 与IO相关的功能，例如序列化
  • JuliaParallel – Julia中的并行编程
  • JuliaWeb – Web栈
  • JuliaCloud - 云提供商

  • JuliaIntervals - 具有区间算术和应用的严格数字

  • JuliaOpt – 优化 (Gitter)
  • JuliaPolyhedra – 多面体计算
  • JuliaSparse – 稀疏矩阵求解器

  • JuliaImages – 图像加工，图像处理
