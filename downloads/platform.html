平台特定说明


Windows 系统

Julia适用于32位和64位Windows 7及更高版本。

  1. 下载适用于您的平台的Windows julia.exe安装程序。32位julia适用于x86和x86_64。64位julia只能在64位Windows（x86_64）上运行。
  2. 运行下载的程序以解压缩julia。
  3. 双击解压缩文件夹中的julia快捷方式以启动julia。

Windows 7 / Windows Server 2012用户还需要安装：

  • 他将 TLS easy_fix 用于包管理器，有关详细信息，请参阅此Discourse主题。
  • Windows Management Framework 3.0或更高版本 ，包括PowerShell 3.0或更高版本。

Windows自述文件包含有关的依赖项的信息。

通过删除 ％HOME％/.julia 中提取的目录和packages目录来执行卸载。如果您还想删除首选项文件，请删除 ％HOME％/.juliarc.jl 和 ％HOME％/.julia_history。


macOS 系统

在Mac上，提供了Julia-<version>.dmg文件，其中包含Julia.app。安装与任何其他Mac软件相同 - 将Julia-<version>.app复制到硬盘驱动器（任何位置）或从磁盘映像运行。Julia在macOS 10.8和更高版本上运行。

通过删除 ~/.julia 中的Julia.app和packages目录来卸载Julia。多个Julia.app二进制文件可以共存而不会相互干扰。如果您还想删除首选项文件，请删除~/.juliarc.jl。

从下载的.dmg安装Julia的替代方法是使用Homebrew包管理器。在您的终端中，运行 

                        
    brew cask install julia


Linux 和 FreeBSD 系统

强烈建议使用下载页面中的官方通用二进制文件在Linux和FreeBSD上安装Julia。


通用二进制文件

通用Linux和FreeBSD二进制文件不需要任何特殊的安装步骤，但您需要确保您的系统可以找到julia可执行文件。首先，将从下载页面下载的.tar.gz文件解压缩到计算机上的文件夹中。要运行Julia，您可以执行以下任何操作：

  • 在系统PATH上的文件夹中创建指向julia的符号链接。
  • 将Julia的bin文件夹添加到系统PATH环境变量中。
  • 使用其完整路径调用julia可执行文件，如 /bin/julia。

例如，要在 /usr/local/bin 文件夹中创建指向 julia 的符号链接，可以执行以下操作： 

                        
    sudo ln -s /bin/julia /usr/local/bin/julia

在某些Linux发行版上，您可能需要使用 /usr/local/bin 以外的文件夹。要检查PATH上的文件夹，可以运行echo $PATH。


特俗的分发包

以下特定于发行版的包是社区贡献的。他们可能不会使用正确版本的Julia依赖项，也不会包含官方二进制文件附带的重要补丁。 通常，只有在下载页面上的官方通用二进制文件中可以重现错误报告时，才会接受错误报告。

随着更多的Linux发行版开始内置julia，这里将作出说明。如果此处没有列出您的Linux发行版，您仍然可以通过从源代码构建来运行julia。有关更多详细信息，请参阅Julia README。


Fedora/RHEL/CentOS/SL/OEL

为Fedora，RHEL，CentOS，Scientific Linux 和 Oracle Enterprise Linux 系统提供了Copr存储库，以允许自动更新到最新的稳定版Julia。

如果您使用的是RHEL，CentOS，Scientific Linux或Oracle Enterprise Linux（版本5或更高版本），请首先为您的分发版本启用EPEL。然后按照以下步骤操作。

如果您使用的是Fedora（版本19或更高版本），请直接运行： 

                        
    sudo dnf copr enable nalimilan/julia
    sudo yum install julia

如果您使用的是CentOS（版本7或更高版本），请直接运行： 

                        
    sudo yum-config-manager --add-repo https://copr.fedorainfracloud.org/coprs/nalimilan/julia/repo/epel-7/nalimilan-julia-epel-7.repo
    sudo yum install julia

如果dnf和yum-config-manager都不适用于您的发行版，请从Copr网页下载相关的.repo文件，将其复制到/etc/yum.repos，然后运行第二个命令。

请注意，Fedora指南建议不要将新的发布版本上传到官方存储库：因此，您的发行版不会提供随后发布的新主要版本的Julia。报告问题时，请确保使用此页面上显示的其中一个Copr存储库来使用最新的可用版本。


Fedora/RHEL/CentOS/SL/OEL nightlies 版本安装说明

为Fedora，RHEL，CentOS，Scientific Linux和Oracle Enterprise Linux系统提供了Copr存储库，以允许自动更新到Julia的最新开发版本。

如果您使用的是RHEL，CentOS，Scientific Linux或Oracle Enterprise Linux（版本5或更高版本），请首先为您的分发版本启用EPEL。然后按照以下步骤操作。

如果您使用的是Fedora（版本19或更高版本），请直接运行： 

                        
    sudo dnf copr enable nalimilan/julia-nightlies
    sudo yum install julia

如果dnf不适用于您的发行版，请从Copr网页下载相关的.repo文件，将其复制到/etc/yum.repos，然后运行第二个命令。

每晚都会构建新版本。如果您已经安装了julia并且想要升级到最新版本，请执行以下操作： 

                        
    sudo yum upgrade julia

FreeBSD Ports

Julia可在Ports Collection中找到。
要从FreeBSD二进制包管理器安装，请运行pkg 

                        
    pkg install julia

要在Ports框架内从源代码构建和安装Julia，请运行 

                        
    cd /usr/ports/lang/julia
    make clean
    make install

卸载Julia

卸载取决于您用于安装Julia的方法。 如果您从包管理器（如apt-get或yum）安装，请使用包管理器删除julia，例如apt-get remove julia或yum remove julia。如果您执行了源代码构建，则可以通过删除julia源文件夹来删除它。 如果您还想删除首选项文件，则它们是~/.julia 和 ~/.juliarc.jl。
